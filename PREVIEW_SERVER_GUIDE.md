# 프리뷰 서버 실행 가이드

## 문제: localhost:4173 연결 거부

프리뷰 서버가 실행되지 않아 연결이 거부되는 경우입니다.

## 해결 방법

### 1. 빌드가 완료되었는지 확인

먼저 빌드가 성공적으로 완료되었는지 확인하세요:

```bash
npm run build
```

빌드가 성공하면 `dist` 폴더가 생성됩니다.

### 2. 프리뷰 서버 실행

빌드가 완료된 후, **새로운 터미널 창**에서 다음 명령어를 실행하세요:

```bash
npm run preview
```

### 3. 서버 실행 확인

프리뷰 서버가 시작되면 다음과 같은 메시지가 표시됩니다:

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:4173/
  ➜  Network: use --host to expose
```

### 4. 브라우저에서 접속

터미널에 표시된 URL(일반적으로 `http://localhost:4173`)을 브라우저에서 열어주세요.

## 한 번에 빌드 + 프리뷰 실행

빌드와 프리뷰를 한 번에 실행하려면:

```bash
npm run build:test
```

이 명령어는 빌드를 실행한 후 자동으로 프리뷰 서버를 시작합니다.

## 문제 해결

### 포트가 이미 사용 중인 경우

4173 포트가 이미 사용 중이면 Vite가 자동으로 다른 포트(4174, 4175 등)를 사용합니다.
터미널에 표시된 실제 포트 번호를 확인하세요.

### dist 폴더가 없는 경우

`dist` 폴더가 없으면 빌드를 먼저 실행해야 합니다:

```bash
npm run build
```

### 서버가 시작되지 않는 경우

1. **의존성 확인**
   ```bash
   npm install
   ```

2. **포트 충돌 확인**
   - 다른 애플리케이션이 4173 포트를 사용 중일 수 있습니다
   - 다른 포트를 사용하도록 설정할 수 있습니다

3. **빌드 오류 확인**
   ```bash
   npm run build
   ```
   빌드 오류가 있으면 먼저 해결해야 합니다.

## 프리뷰 서버 중지

프리뷰 서버를 중지하려면 터미널에서 `Ctrl + C`를 누르세요.

## 참고

- 프리뷰 서버는 빌드된 `dist` 폴더의 내용을 서빙합니다
- 개발 중에는 `npm run dev`를 사용하세요 (포트 3000)
- 빌드된 결과물을 테스트할 때만 `npm run preview`를 사용하세요
